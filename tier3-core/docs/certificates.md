## Generating Certificates

PKI is owned by Tier 4. Run from the repo root:

```bash
# Generate Root CA (tier4-operations/pki/certs/) + all Tier 3 service certs (tier3-core/certs/)
bash tier4-operations/pki/scripts/generate-certs.sh --verbose

# Dry-run (shows what would be generated without writing files)
bash tier4-operations/pki/scripts/generate-certs.sh --dry-run
```

The script writes the Root CA to `tier4-operations/pki/certs/` and copies
`root-ca.pem` into `tier3-core/certs/` alongside the service certs so containers
can mount everything from a single local directory.

## Certificate Paths (Runtime)

All paths are relative to `tier3-core/`. Root CA authority lives in `tier4-operations/pki/certs/`;
`root-ca.pem` is copied into `tier3-core/certs/` by `generate-certs.sh` for container use.

| Service | Certificate | Key | CA |
|---------|-------------|-----|----|
| Logstash (server) | `certs/logstash.pem` | `certs/logstash-key.pem` | `certs/root-ca.pem` |
| OpenSearch (node) | `certs/node.pem` | `certs/node-key.pem` | `certs/root-ca.pem` |
| OpenSearch (admin) | `certs/admin.pem` | `certs/admin-key.pem` | `certs/root-ca.pem` |
| OpenSearch (dashboards) | `certs/dashboard.pem` | `certs/dashboard-key.pem` | `certs/root-ca.pem` |

All `*-key.pem` files are `chmod 600` and **gitignored** (root `.gitignore` covers `*.pem`).
The CA public cert (`root-ca.pem`) is safe to distribute to all Tier 1/2 nodes.

## TLS Configuration

- **Minimum TLS version:** 1.3 on all services
- **Cipher suites:** OpenSSL defaults for TLS 1.3 (AES-256-GCM, CHACHA20)
- **Certificate verification:** `verify_mode => peer` on Logstash input
  (client cert required from all syslog-ng forwarders)
- **OpenSearch:** `enforce_hostname_verification: true` for node transport

## Certificate Rotation

1. Regenerate all certs from repo root: `bash tier4-operations/pki/scripts/generate-certs.sh --verbose`
2. Restart affected tier3-core services: `docker compose restart <service>`
3. If rotating the Root CA, all tiers must be updated in the same change:
   - Regenerate all certs: `bash tier4-operations/pki/scripts/generate-certs.sh --verbose`
   - Re-issue Tier 1 client cert: `make deploy-skip-rules` in `tier1-perimeter/`
   - Restart all SIEM services
4. Verify: `openssl verify -CAfile tier3-core/certs/root-ca.pem tier3-core/certs/<service>.pem`

---

## Security Notes

- Root CA key (`root-ca-key.pem`) must never be stored on the SIEM host in plaintext
- Recommended: store Root CA key in SOPS-encrypted file or hardware token
- `tier4-operations/pki/certs/root-ca.srl` (serial counter) and `root-ca.pem` are the only cert artifacts tracked in Git
- All `*.pem` and `*-key.pem` files in `tier3-core/certs/` are **gitignored** — runtime only
