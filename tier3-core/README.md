# Tier 3 — Core (Analytics & Detection)

> **Role:** Centralised analytics, storage, and detection engine for the SURU
> platform. Receives mTLS-authenticated telemetry from Tier 1 (pfSense/OPNsense)
> and Tier 2 (syslog-ng forwarder), normalises every event to **Elastic Common
> Schema (ECS) v8**, indexes into OpenSearch, and surfaces alerts through
> OpenSearch Dashboards.
>
> Platform health/monitoring previously lived under `tier3-core/monitoring/`
> and has moved to [`tier4-operations/monitoring/`](../tier4-operations/) as
> part of the Tier 4 operations / control-plane split.

---

## Prerequisites

| Requirement | Details |
|-------------|---------|
| Host OS | Linux x86-64 (Ubuntu 22.04 LTS / Debian 12 recommended) |
| Docker | 24.x+ with Compose v2 plugin (`docker compose`) |
| RAM | 8 GB minimum · 16 GB recommended for production |
| Disk | 50 GB minimum for index data (`datalake/opensearch/`) |
| Ports open | 5140/tcp (syslog TLS in), 9200/tcp (OpenSearch), 5601/tcp (Dashboards), 5044/tcp (Beats/Logstash) |
| SURU Root CA | Generated by `scripts/generate-certs.sh` — **run this first** |
| Local tools | `docker`, `openssl`, `make`, optionally `shellcheck`, `yamllint` |

---

## Quick Start

```bash
# 1. Generate SURU Root CA + all service certificates
bash scripts/generate-certs.sh

# 2. Configure environment
cp .env.example .env
$EDITOR .env   # fill in passwords and hostnames

# 3. Bring up the full SIEM stack
bash scripts/start-all.sh

# 4. (Optional) Import default OpenSearch dashboards
bash scripts/import-dashboards.sh
```

See [`docs/deployment.md`](./docs/deployment.md) for the full step-by-step guide.

## Subsystem Overview

| Subsystem | Location | Purpose |
|-----------|----------|---------|
| **OpenSearch** | `datalake/opensearch/` | Primary data store — ECS-indexed events, full-text search, aggregations |
| **OpenSearch Dashboards** | `datalake/opensearch/` | Visualisation, saved searches, alerting UI |
| **Logstash (OPNsense)** | `ingestion/logstash/` + `config/logstash-opnsense/` | OPNsense syslog UDP ingest pipeline |
| **Logstash (pfSense)** | `ingestion/logstash/` + `config/logstash-pfsense/` | pfSense mTLS syslog TCP ingest pipeline |
| **Monitoring** | `monitoring/` | Prometheus metrics + Grafana dashboards for SIEM health |

## Security Notes

- `.env` and all `*-key.pem` files are **gitignored** — never commit them
- All inter-service communication uses **mTLS** (mutual TLS 1.3)
- OpenSearch security plugin enforces **RBAC** — see `config/opensearch/security/`
- Logstash verifies client certificates against the SURU Root CA before accepting any event
- Audit logs for all privileged actions are forwarded to `suru-audit-*` index
